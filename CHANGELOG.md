# Changelog

## 2026-07-19

- Published the updated twelve-file workflow version 6 payload, including the
  cross-platform PowerShell launchers.
- Split normal private GitHub source synchronization from strict
  public-readiness review so private pushes no longer block on operational
  metadata, while strict public and sanitized-backup audits still review it.

  constants, added line numbers to local audit findings, and kept known token
  formats blocked.

## 2026-07-02

- Added automatic sanitized GitHub backup guidance to the new project setup
  workflow.
- Clarified that GitHub backups verify complete sanitized snapshots while using
  incremental Git pushes.
- Added public-facing changelog guidance for projects using the setup workflow.

## 2026-07-03

- Added strict target-scope guidance so `$new-project-setup` updates only the
  explicitly targeted or active chat/session project.
- Clarified that dirty or untracked local files do not block GitHub backup of a
  committed sanitized snapshot.
- Clarified that source-project skill maintenance syncs behavior changes to the
  installed runtime skill during the same user-invoked workflow.

## 2026-07-06

- Added prerequisite bootstrap guidance so new users can have Codex check and,
  with approval, install Git and the GitHub CLI, authenticate GitHub, and
  configure Git identity before project setup continues.
- Added autonomous work-package guidance so Codex can complete bounded
  multi-step development work with fewer routine user checkpoints.
- Clarified that `$new-project-setup` invocations in non-source projects apply
  the installed workflow to that project only and must not modify the skill
  source or installed runtime skill.
- Clarified that a bare `$new-project-setup` invocation should run install/sync
  for the resolved target project instead of acting as context-only loading.
- Strengthened bare invocation handling so "loaded only/no actions" is
  explicitly invalid when a target project can be resolved safely.
- Added the mandatory bare-invocation behavior to the skill description so
  activation metadata also tells Codex to run install/sync.
- Clarified that questions about `$new-project-setup` are consultation-only and
  do not authorize edits unless the user also explicitly asks to apply, update,
  sync, or maintain the workflow.
